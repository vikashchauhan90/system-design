## Validate IP Address

**Description:** Given a string `queryIP`, return "IPv4" if the input is a valid IPv4 address, "IPv6" if it is a valid IPv6 address, or "Neither" otherwise.

A valid IPv4 address has the form `x1.x2.x3.x4` where each `xi` is a decimal integer in the range [0,255] and must not contain leading zeros (except the single digit "0"). A valid IPv6 address has the form `x1:x2:...:x8` where each `xi` is 1 to 4 hexadecimal digits (0-9, a-f, A-F). Leading zeros are allowed in IPv6 segments.

**Examples:**
```
Input: queryIP = "172.16.254.1"
Output: "IPv4"
Explanation: Valid IPv4

Input: queryIP = "2001:0db8:85a3:0:0:8A2E:0370:7334"
Output: "IPv6"
Explanation: Valid IPv6

Input: queryIP = "256.256.256.256"
Output: "Neither"
Explanation: Each IPv4 segment must be 0..255
```

#### Constraints
- `queryIP` contains only English letters, digits, and the characters `.` and `:`.
- An address cannot contain both `.` and `:` separators.

### Approach

Decide the candidate type by the presence of separators: if the string contains `.` and not `:`, validate as IPv4; if it contains `:` and not `.`, validate as IPv6. Otherwise return "Neither". Validation rules are applied strictly (no empty segments, correct counts of segments, allowed characters, value ranges, no leading zeros for IPv4 segments).

### Pseudocode

```
FUNCTION ValidateIP(queryIP):
    IF queryIP CONTAINS '.' AND NOT CONTAINS(':'):
        RETURN ValidateIPv4(queryIP)
    ELSE IF queryIP CONTAINS ':' AND NOT CONTAINS('.'):
        RETURN ValidateIPv6(queryIP)
    ELSE:
        RETURN "Neither"

FUNCTION ValidateIPv4(ip):
    parts = SPLIT(ip, '.')
    IF LENGTH(parts) != 4: RETURN "Neither"

    FOR part IN parts:
        IF part IS EMPTY: RETURN "Neither"
        IF LENGTH(part) > 1 AND part[0] == '0': RETURN "Neither" // leading zero
        FOR ch IN part:
            IF ch NOT IN '0'..'9': RETURN "Neither"
        END FOR
        val = PARSE_INT(part)
        IF val < 0 OR val > 255: RETURN "Neither"
    END FOR

    RETURN "IPv4"

FUNCTION ValidateIPv6(ip):
    parts = SPLIT(ip, ':')
    IF LENGTH(parts) != 8: RETURN "Neither"

    FOR part IN parts:
        IF part IS EMPTY: RETURN "Neither"
        IF LENGTH(part) < 1 OR LENGTH(part) > 4: RETURN "Neither"
        FOR ch IN part:
            IF ch NOT IN hex digits [0-9a-fA-F]: RETURN "Neither"
        END FOR
    END FOR

    RETURN "IPv6"
```

```csharp
// C# implementation following the above rules
public static string ValidIPAddress(string IP)
{
    if (IP.Contains('.') && !IP.Contains(':')) return ValidateIPv4(IP);
    if (IP.Contains(':') && !IP.Contains('.')) return ValidateIPv6(IP);
    return "Neither";
}

private static string ValidateIPv4(string ip)
{
    var parts = ip.Split('.');
    if (parts.Length != 4) return "Neither";

    foreach (var p in parts)
    {
        if (p.Length == 0 || p.Length > 3) return "Neither";
        if (p.Length > 1 && p[0] == '0') return "Neither"; // leading zero
        foreach (var ch in p) if (!char.IsDigit(ch)) return "Neither";
        if (!int.TryParse(p, out int val)) return "Neither";
        if (val < 0 || val > 255) return "Neither";
    }
    return "IPv4";
}

private static string ValidateIPv6(string ip)
{
    var parts = ip.Split(':');
    if (parts.Length != 8) return "Neither";

    foreach (var p in parts)
    {
        if (p.Length == 0 || p.Length > 4) return "Neither";
        foreach (var ch in p)
        {
            bool isHexDigit = (ch >= '0' && ch <= '9') ||
                              (ch >= 'a' && ch <= 'f') ||
                              (ch >= 'A' && ch <= 'F');
            if (!isHexDigit) return "Neither";
        }
    }
    return "IPv6";
}
```

### Visualization (step-by-step)

Example 1: `"172.16.254.1"` (candidate IPv4)
- Split into ["172","16","254","1"] → 4 parts ✓
- Each part non-empty, no leading zeros, digits-only → parse: 172,16,254,1 all within 0..255 → return "IPv4"

Example 2: `"2001:0db8:85a3:0:0:8A2E:0370:7334"` (candidate IPv6)
- Split into 8 parts, each length 1..4 and characters in 0-9/a-f/A-F → return "IPv6"

Example 3: `"256.256.256.256"`
- Split into 4 parts → digits-only, but parsed values 256 > 255 → return "Neither"


### Complexity
- **Time Complexity :** `O(n)` where `n` is the length of `queryIP` — each character is examined at most a constant number of times.
- **Space Complexity :** `O(n)` due to splitting into parts (or `O(1)` extra if validated with a single scan without creating split arrays).
