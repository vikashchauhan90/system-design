
## Compress String

**Description:** Implement basic string compression using counts of repeated characters. For example, the string "aabcccccaaa" would become "a2b1c5a3". If the compressed string is not smaller than the original, return the original string.

**Examples:**
```
Input: s = "aabcccccaaa"
Output: "a2b1c5a3"
Explanation: 'a' repeated 2, 'b' 1, 'c' 5, 'a' 3

Input: s = "abcdef"
Output: "abcdef"
Explanation: Compression would be "a1b1c1d1e1f1" which is longer than original

Input: s = "a"
Output: "a"
Explanation: Single-character string returns original
```

#### Constraints
- Input `s` is a string containing ASCII letters (or general characters) and may be empty.
- Preserve characters' relative order; only consecutive runs are compressed.

### Approach (Run-Length Encoding)

We scan the string once and accumulate counts for consecutive identical characters. Append the character followed by its count to a builder. Optionally, compute the compressed length first to avoid building when compression won't shorten the string.

### Brute Force / Straightforward

**Pseudocode:**

```
FUNCTION CompressString(s):
    IF s IS NULL OR LENGTH(s) <= 1:
        RETURN s

    // Use a mutable builder to accumulate compressed output
    compressed = NEW STRING_BUILDER()

    current_char = s[0]
    count = 1

    FOR i FROM 1 TO LENGTH(s)-1:
        IF s[i] == current_char:
            count = count + 1
        ELSE:
            compressed.APPEND(current_char)
            compressed.APPEND(TO_STRING(count))
            current_char = s[i]
            count = 1
        END IF
    END FOR

    // Append the final run
    compressed.APPEND(current_char)
    compressed.APPEND(TO_STRING(count))

    result = compressed.TO_STRING()
    IF LENGTH(result) >= LENGTH(s):
        RETURN s
    ELSE:
        RETURN result
    END IF
END FUNCTION
```

```csharp
// C# example (straightforward implementation)
public static string CompressString(string s)
{
    if (string.IsNullOrEmpty(s) || s.Length == 1) return s;

    var sb = new StringBuilder();
    char current = s[0];
    int count = 1;

    for (int i = 1; i < s.Length; i++)
    {
        if (s[i] == current)
        {
            count++;
        }
        else
        {
            sb.Append(current);
            sb.Append(count);
            current = s[i];
            count = 1;
        }
    }

    sb.Append(current);
    sb.Append(count);

    var compressed = sb.ToString();
    return compressed.Length >= s.Length ? s : compressed;
}
```

### Visualization (step-by-step)

Example: `s = "aabcccccaaa"`
- Start: current='a', count=1, compressed=""
- i=1: s[1]='a' == current → count=2
- i=2: s[2]='b' != current → compressed="a2"; current='b', count=1
- i=3: s[3]='c' != current → compressed="a2b1"; current='c', count=1
- i=4: s[4]='c' == current → count=2
- i=5: s[5]='c' == current → count=3
- i=6: s[6]='c' == current → count=4
- i=7: s[7]='c' == current → count=5
- i=8: s[8]='a' != current → compressed="a2b1c5"; current='a', count=1
- i=9: s[9]='a' == current → count=2
- i=10: s[10]='a' == current → count=3
- End: append final run → compressed="a2b1c5a3" → compare lengths (11 vs 10) → compressed shorter → return compressed

Visual build: ["" -> "a2" -> "a2b1" -> "a2b1c5" -> "a2b1c5a3"]

### Optimized Variant (optional)

If you expect many short runs, you can first compute the length of the compressed string without building it. If the computed compressed length is >= original length, return original immediately. This avoids work and memory allocation when compression is useless.

Pseudocode for compressed length calculation:

```
FUNCTION CompressedLength(s):
    IF s IS NULL OR LENGTH(s) == 0: RETURN 0
    length = 0
    count = 1
    FOR i FROM 1 TO LENGTH(s)-1:
        IF s[i] == s[i-1]:
            count = count + 1
        ELSE:
            length = length + 1 + LENGTH(TO_STRING(count)) // 1 for char, digits for count
            count = 1
        END IF
    END FOR
    length = length + 1 + LENGTH(TO_STRING(count))
    RETURN length
END FUNCTION
```

Use this to decide whether to build the compressed string.


### Complexity
- **Time Complexity :** `O(n)` — single scan to count runs (or two scans for the length-then-build variant).
- **Space Complexity :** `O(n)` — worst-case compressed string (or builder) may be proportional to input length. In-place compression is possible only with mutable buffers and given constraints, but general approach uses additional output buffer.
