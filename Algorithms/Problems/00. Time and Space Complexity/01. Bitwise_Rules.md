### Overview
Below is a compact, practical rule set for the five bitwise operators you listed. Each operator has its algebraic identities, common bit tricks, and algorithmic uses. Read the table first for a quick comparison, then use the rules and tricks section for implementation patterns.

### Comparison table of operators and key attributes

| **Operator** | **Symbol** | **Effect on bits** | **Algebraic properties** | **Common use** |
|---|---:|---|---|---|
| **bitwise AND** | `&` | result bit is 1 only if both input bits are 1 | **Commutative**, **Associative**, **Idempotent**: `x & x = x` | masking, clearing bits, testing flags |
| **bitwise OR** | `|` | result bit is 1 if at least one input bit is 1 | **Commutative**, **Associative**, **Idempotent**: `x | x = x` | setting bits, combining flags |
| **bitwise XOR** | `^` | result bit is 1 if input bits differ | **Commutative**, **Associative**, **Self inverse**: `x ^ x = 0` | toggling bits, parity, difference |
| **right shift** | `>>` | moves bits toward LSB, fills high bits depending on type | not commutative with other ops, repeated shift divides by two for unsigned | divide by powers of two, extract low-order bits |
| **left shift** | `<<` | moves bits toward MSB, fills low bits with zeros | repeated shift multiplies by two for values that do not overflow | multiply by powers of two, create masks |

### Algebraic identities and rules

- **Zero and identity elements**
  - `x & 0 = 0`
  - `x | 0 = x`
  - `x ^ 0 = x`
  - `x & ~0 = x`
  - `x | ~0 = ~0`

- **Idempotence and involution**
  - `x & x = x`
  - `x | x = x`
  - `x ^ x = 0`
  - `x ^ 0 = x`

- **Distributive laws**
  - `x & (y | z) = (x & y) | (x & z)`
  - `x | (y & z) = (x | y) & (x | z)`

- **De Morgan style complements**
  - `~(x & y) = ~x | ~y`
  - `~(x | y) = ~x & ~y`

- **XOR properties**
  - XOR is its own inverse, so `x ^ a ^ a = x`
  - XOR with mask toggles bits, so `x ^ mask` flips bits set in mask

- **Combining AND, OR, XOR**
  - `x ^ y = (x | y) & ~(x & y)` gives XOR from OR and AND
  - `x | y = (x & y) ^ (x ^ y)` alternative decomposition

### Common bit tricks used in algorithms

- **Test if bit k is set**
  - `((x >> k) & 1) == 1` returns true when bit k is 1

- **Set bit k**
  - `x = x | (1 << k)` sets bit k to 1

- **Clear bit k**
  - `x = x & ~(1 << k)` sets bit k to 0

- **Toggle bit k**
  - `x = x ^ (1 << k)` flips bit k

- **Extract lowest set bit**
  - `low = x & -x` returns a mask with only the least significant 1
  - This uses two's complement negative

- **Turn off lowest set bit**
  - `x = x & (x - 1)` clears the least significant 1
  - Useful for counting set bits in a loop

- **Check power of two**
  - `x > 0 && (x & (x - 1)) == 0` true when x is a power of two

- **Count bits loop**
  - while `x != 0` do `x &= x - 1` and increment counter
  - Each iteration removes one 1 bit

- **Swap two bits at positions i and j**
  - `if (((x >> i) & 1) != ((x >> j) & 1)) x ^= (1 << i) | (1 << j)`

- **Create mask of n low bits**
  - `mask = (1 << n) - 1` produces n ones in low bits when n < word size

- **Isolate bits in range [l, r]**
  - `mask = ((1 << (r - l + 1)) - 1) << l`
  - `result = (x & mask) >> l`

- **Parity of bits**
  - `parity = x ^ (x >> 1) ^ (x >> 2) ...` reduced by XOR folding
  - Final parity bit indicates odd or even number of 1s

### Shift rules and sign considerations

- **Left shift**
  - `x << k` moves bits left by k positions, inserts zeros on the right
  - For unsigned interpretation, `x << k` multiplies by \(2^{k}\) when no overflow occurs
  - Shifting into the sign bit causes overflow for signed types

- **Right shift**
  - Logical right shift fills high bits with zeros
  - Arithmetic right shift replicates the sign bit for signed types, preserving sign
  - In many languages `>>` on signed integers is arithmetic shift, `>>>` is logical shift
  - For unsigned interpretation, `x >> k` divides by \(2^{k}\) rounding toward zero

- **Shift bounds**
  - Shifting by an amount greater than or equal to the word size is undefined or implementation defined in many languages
  - Always ensure shift amount is within valid range

### Examples and pitfalls

- **XOR of equal numbers**
  - `a ^ a = 0` always true

- **Masking example**
  - To keep only low 5 bits do `x & 0x1F`

- **Sign bit swap caution**
  - Swapping or shifting into the sign bit can change sign and produce large magnitude changes
  - When working with bit patterns rather than signed arithmetic, use unsigned types if available

- **Overflow and undefined behavior**
  - Left shifting a 1 into the sign bit for signed integers may be undefined
  - Avoid relying on behavior when shifting near the word size limit

 
 -- **Mask definition:** a mask is an integer where 1 means “keep” and 0 means “ignore/clear” for AND operations.
