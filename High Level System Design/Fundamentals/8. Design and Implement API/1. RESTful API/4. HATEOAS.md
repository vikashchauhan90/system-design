# HATEOAS & Richardson Maturity Model

## Introduction
**HATEOAS** (Hypermedia as the Engine of Application State) is a constraint of REST that makes APIs self‑discoverable. Responses include hyperlinks to possible actions. The **Richardson Maturity Model** classifies APIs into four levels (0–3). Level 3 is true REST with HATEOAS.

### Richardson Maturity Model

| Level | Description | Example |
|-------|-------------|---------|
| 0 | One URI, one method (usually POST) for all actions. | `POST /api` with action parameter. |
| 1 | Multiple URIs (resources) but single method. | `POST /orders`, `POST /customers` |
| 2 | Uses HTTP verbs (GET, POST, PUT, DELETE) and status codes. | `GET /orders/123`, `DELETE /orders/123` |
| 3 | Level 2 + HATEOAS (hypermedia links). | Response contains `"links": [{ "rel": "self", "href": "/orders/123" }]` |

### HATEOAS Best Practices
- Use **relative URLs** (e.g., `/orders/123`) instead of absolute (`https://api.example.com/orders/123`) to avoid coupling to hostname.
- Include a `self` link for every resource.
- Add links for possible state transitions (e.g., `cancel`, `pay`) based on resource state.
- Use standard link relations (`self`, `next`, `prev`, `collection`, `item`) when possible.
- Document custom relations in your API documentation.

### C# Implementation (ASP.NET Core)

**Link Model**
```csharp
public class Link
{
    public string Href { get; set; }
    public string Rel { get; set; }
    public string Method { get; set; }
}
```

**Resource with Links**
```csharp
public class Order
{
    public int Id { get; set; }
    public decimal Total { get; set; }
    public string Status { get; set; }
    public List<Link> Links { get; set; } = new();
}
```

**Controller using IUrlHelper for Relative URLs**
```csharp
[ApiController]
[Route("api/orders")]
public class OrdersController : ControllerBase
{
    private readonly IUrlHelper _urlHelper;
    public OrdersController(IUrlHelperFactory factory, IActionContextAccessor accessor)
    {
        _urlHelper = factory.GetUrlHelper(accessor.ActionContext);
    }

    [HttpGet("{id}")]
    public IActionResult GetOrder(int id)
    {
        var order = GetOrderFromDb(id); // fetch your domain object
        order.Links = new List<Link>
        {
            new Link { Rel = "self", Href = _urlHelper.Link("GetOrder", new { id }), Method = "GET" },
            new Link { Rel = "cancel", Href = _urlHelper.Link("CancelOrder", new { id }), Method = "DELETE" },
            new Link { Rel = "pay", Href = _urlHelper.Link("PayOrder", new { id }), Method = "POST" }
        };
        return Ok(order);
    }

    [HttpDelete("{id}", Name = "CancelOrder")]
    public IActionResult CancelOrder(int id) { ... }

    [HttpPost("{id}/pay", Name = "PayOrder")]
    public IActionResult PayOrder(int id) { ... }
}
```

**Response Example**
```json
{
  "id": 123,
  "total": 99.99,
  "status": "pending",
  "links": [
    { "rel": "self", "href": "/api/orders/123", "method": "GET" },
    { "rel": "cancel", "href": "/api/orders/123", "method": "DELETE" },
    { "rel": "pay", "href": "/api/orders/123/pay", "method": "POST" }
  ]
}
```

**Benefits:** Clients no longer need to hardcode URLs; they follow links. The API can evolve without breaking clients as long as link relations remain consistent.
