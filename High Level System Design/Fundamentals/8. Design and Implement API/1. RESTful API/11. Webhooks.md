## 1. What are Webhooks?

A **webhook** is an HTTP callback triggered by an event in a system. The provider (your API) sends a POST request to a URL provided by the consumer (client) when an event occurs. Webhooks enable real‑time notifications without polling.

**Common use cases:**
- Payment gateways notifying merchants of successful transactions.
- CI/CD pipelines triggering deployments.
- E‑commerce platforms sending order updates to fulfillment systems.

---

## 2. Webhook Lifecycle

1. **Registration** – Consumer registers a webhook URL with the provider, including authentication credentials (API key, JWT) and optionally a shared secret.
2. **Verification** – Provider may send a challenge (e.g., `POST` with a `challenge` field) to verify ownership of the URL.
3. **Event Delivery** – When an event occurs, the provider constructs a payload, signs it, and sends it to the registered URL.
4. **Acknowledgement** – Consumer returns a `2xx` status code to acknowledge receipt. Non‑`2xx` responses trigger retries.
5. **Deactivation** – Consumer can delete or update the webhook registration.

---

## 3. Secure Webhook Registration (Token-Based Authentication)

To prevent unauthorised registrations, the provider must authenticate the client using a token (e.g., API key, Bearer JWT). The client also provides a **shared secret** (or the provider generates one) that will be used later to sign webhook payloads.

### Registration Request Example

**Client sends:**
```http
POST /api/webhooks/register
Authorization: Bearer <access_token>
Content-Type: application/json

{
  "url": "https://consumer.example.com/webhooks/orders",
  "events": ["order.created", "order.paid"],
  "secret": "optional-custom-shared-secret"
}
```

### Server-Side Implementation (C#)

**Model:**
```csharp
public class WebhookRegistrationRequest
{
    public string Url { get; set; }
    public List<string> Events { get; set; }
    public string? Secret { get; set; }
}

public class WebhookRegistration
{
    public int Id { get; set; }
    public string Url { get; set; }
    public string ClientId { get; set; }
    public List<string> Events { get; set; }
    public string SharedSecret { get; set; }
    public DateTime CreatedAt { get; set; }
    public bool IsActive { get; set; } = true;
}
```

**Registration Controller (with authentication)**
```csharp
[Authorize] // JWT or API key authentication
[ApiController]
[Route("api/webhooks")]
public class WebhookRegistrationController : ControllerBase
{
    private readonly IWebhookStore _store;

    [HttpPost("register")]
    public async Task<IActionResult> Register([FromBody] WebhookRegistrationRequest request)
    {
        // Enforce HTTPS in production
        if (!Uri.TryCreate(request.Url, UriKind.Absolute, out var uri) || 
            (uri.Scheme != Uri.UriSchemeHttps && !IsDevelopment()))
            return BadRequest("Only HTTPS URLs are allowed.");

        // Generate a cryptographically random secret if not provided
        var secret = string.IsNullOrWhiteSpace(request.Secret) 
            ? GenerateSharedSecret() 
            : request.Secret;

        var registration = new WebhookRegistration
        {
            Url = request.Url,
            ClientId = User.FindFirstValue(ClaimTypes.NameIdentifier),
            Events = request.Events,
            SharedSecret = secret,
            CreatedAt = DateTime.UtcNow
        };
        await _store.SaveAsync(registration);

        // Optional: Send verification challenge
        await SendVerificationChallenge(registration);

        return Ok(new { registration.Id, secret });
    }

    private string GenerateSharedSecret()
    {
        var bytes = new byte[32];
        RandomNumberGenerator.Fill(bytes);
        return Convert.ToBase64String(bytes);
    }

    private async Task SendVerificationChallenge(WebhookRegistration reg)
    {
        var challenge = Guid.NewGuid().ToString();
        var payload = new { challenge, registrationId = reg.Id };
        using var httpClient = new HttpClient();
        var response = await httpClient.PostAsJsonAsync(reg.Url, payload);
        // If success, mark as verified; otherwise retry later.
    }
}
```

**Best Practices for Registration:**
- Always use HTTPS for webhook URLs.
- Generate a strong shared secret (minimum 32 bytes) on the server if not provided.
- Store the secret encrypted at rest.
- Limit the number of webhooks per client.

## 4. Securing Webhook Delivery: Signatures & Encryption

The provider must prove that the webhook payload is authentic (not tampered) and originated from the expected source. The standard method is **HMAC signature** using the shared secret, sent in the `X-Webhook-Signature` header.

### 4.1 Signature Header (`X-Webhook-Signature`)

The `X-Webhook-Signature` header contains a signature computed over the request body (or a canonical representation) plus an optional timestamp. Format:

```
X-Webhook-Signature: sha256=base64(hmac)
```

**Optional additional headers:**
- `X-Webhook-Timestamp`: Unix timestamp (seconds) to prevent replay attacks.
- `X-Webhook-Id`: Unique identifier for idempotency.

### 4.2 Signature Generation (Provider Side)

**Algorithm:** `HMAC-SHA256` of `timestamp + "." + raw_body` (or just the raw body if no timestamp). Using timestamp is strongly recommended.

**C# Provider Implementation (sending webhook)**
```csharp
public class WebhookDeliveryService
{
    public async Task SendWebhookAsync(WebhookRegistration registration, object eventPayload)
    {
        var jsonPayload = JsonSerializer.Serialize(eventPayload);
        var timestamp = DateTimeOffset.UtcNow.ToUnixTimeSeconds().ToString();
        var signature = ComputeSignature(jsonPayload, timestamp, registration.SharedSecret);

        using var client = new HttpClient();
        var request = new HttpRequestMessage(HttpMethod.Post, registration.Url)
        {
            Content = new StringContent(jsonPayload, Encoding.UTF8, "application/json")
        };
        request.Headers.Add("X-Webhook-Signature", $"sha256={signature}");
        request.Headers.Add("X-Webhook-Timestamp", timestamp);
        request.Headers.Add("X-Webhook-Id", Guid.NewGuid().ToString());

        var response = await client.SendAsync(request);
        if (!response.IsSuccessStatusCode)
        {
            // Queue for retry (see section 5)
        }
    }

    private string ComputeSignature(string payload, string timestamp, string secret)
    {
        var message = timestamp + "." + payload;
        using var hmac = new HMACSHA256(Encoding.UTF8.GetBytes(secret));
        var hash = hmac.ComputeHash(Encoding.UTF8.GetBytes(message));
        return Convert.ToBase64String(hash);
    }
}
```

### 4.3 Signature Validation (Consumer Side)

The consumer MUST verify the signature before processing the webhook.

**C# Consumer Implementation (ASP.NET Core endpoint)**
```csharp
[HttpPost("webhooks/orders")]
public async Task<IActionResult> HandleOrderWebhook()
{
    // 1. Read required headers
    if (!Request.Headers.TryGetValue("X-Webhook-Signature", out var signatureHeader) ||
        !Request.Headers.TryGetValue("X-Webhook-Timestamp", out var timestampHeader))
        return BadRequest("Missing signature or timestamp");

    var signature = signatureHeader.ToString().Replace("sha256=", "");
    var timestamp = timestampHeader.ToString();

    // 2. Read raw body (must be buffered)
    var rawBody = await ReadRawBodyAsync();

    // 3. Retrieve the shared secret for this webhook endpoint
    //    (e.g., based on URL path or a client ID from the request)
    var secret = await _secretStore.GetSecretForWebhookUrl(Request.Path);

    // 4. Compute expected signature
    var expectedSignature = ComputeSignature(rawBody, timestamp, secret);

    // 5. Constant-time comparison to prevent timing attacks
    if (!CryptographicOperations.FixedTimeEquals(
        Convert.FromBase64String(signature), 
        Convert.FromBase64String(expectedSignature)))
    {
        return Unauthorized("Invalid signature");
    }

    // 6. Check timestamp freshness (e.g., within 5 minutes)
    var currentTime = DateTimeOffset.UtcNow.ToUnixTimeSeconds();
    if (Math.Abs(currentTime - long.Parse(timestamp)) > 300)
        return BadRequest("Webhook too old");

    // 7. Deserialize and process
    var payload = JsonSerializer.Deserialize<OrderEvent>(rawBody);
    await _orderProcessor.ProcessAsync(payload);

    return Ok();
}

private string ComputeSignature(string payload, string timestamp, string secret)
{
    var message = timestamp + "." + payload;
    using var hmac = new HMACSHA256(Encoding.UTF8.GetBytes(secret));
    var hash = hmac.ComputeHash(Encoding.UTF8.GetBytes(message));
    return Convert.ToBase64String(hash);
}

private async Task<string> ReadRawBodyAsync()
{
    Request.EnableBuffering();
    using var reader = new StreamReader(Request.Body, leaveOpen: true);
    var body = await reader.ReadToEndAsync();
    Request.Body.Position = 0;
    return body;
}
```

**Important:** Always use `CryptographicOperations.FixedTimeEquals` to avoid timing attacks when comparing signatures.

### 4.4 Optional Payload Encryption

If webhook payloads contain sensitive data, encrypt them before sending. A standard approach is **JWE** (JSON Web Encryption) or AES‑GCM. For simplicity, you can encrypt the entire body using a symmetric key derived from the shared secret.

**Provider: Encrypt payload before signing**
```csharp
public string EncryptPayload(string plainText, string secretKey)
{
    using var aes = Aes.Create();
    aes.Key = Convert.FromBase64String(secretKey); // secret must be 32 bytes for AES-256
    aes.GenerateIV();
    var encryptor = aes.CreateEncryptor();
    var plainBytes = Encoding.UTF8.GetBytes(plainText);
    var cipherBytes = encryptor.TransformFinalBlock(plainBytes, 0, plainBytes.Length);
    var result = new byte[aes.IV.Length + cipherBytes.Length];
    Buffer.BlockCopy(aes.IV, 0, result, 0, aes.IV.Length);
    Buffer.BlockCopy(cipherBytes, 0, result, aes.IV.Length, cipherBytes.Length);
    return Convert.ToBase64String(result);
}
```

Then sign the **encrypted payload** (the ciphertext). The consumer would decrypt after signature validation.

## 5. Retry Mechanism & Exponential Backoff

Webhook delivery may fail due to network issues or consumer downtime. Implement a retry queue with exponential backoff.

**Retry Policy Example:**
- Max retries: 5
- Initial delay: 1 second
- Backoff factor: 2 (1s, 2s, 4s, 8s, 16s)

**C# Retry Queue (using Hangfire or BackgroundService)**
```csharp
public class WebhookRetryService
{
    private readonly IBackgroundJobClient _jobClient;
    public async Task EnqueueRetry(WebhookRegistration reg, object payload, int attempt)
    {
        var delay = TimeSpan.FromSeconds(Math.Pow(2, attempt));
        _jobClient.Schedule(() => SendWebhookAsync(reg, payload, attempt + 1), delay);
    }
}
```

**Response handling:** Consumer should return `2xx` for success, `4xx` for permanent failure (don’t retry), and `5xx` for temporary failure (retry).

## 6. Idempotency for Webhook Events

Network issues may cause duplicate webhook deliveries. Consumers should handle idempotency using the `X-Webhook-Id` header.

**Consumer side:**
```csharp
private readonly IIdempotencyStore _idempotencyStore;

public async Task<IActionResult> HandleOrderWebhook()
{
    var webhookId = Request.Headers["X-Webhook-Id"].ToString();
    if (await _idempotencyStore.IsProcessedAsync(webhookId))
        return Ok(); // Already processed

    // Process the event...

    await _idempotencyStore.MarkProcessedAsync(webhookId);
    return Ok();
}
```

Store processed IDs in a cache (Redis) with TTL = event retention period.

## 7. Best Practices for Webhooks

| Area | Recommendation |
|------|----------------|
| **URL** | Enforce HTTPS only. Validate URL format and domain allowlist. |
| **Authentication** | Use token-based auth (API key, JWT) for registration. |
| **Signature** | Always use `X-Webhook-Signature` with HMAC-SHA256. Include timestamp. |
| **Replay Protection** | Validate timestamp freshness (e.g., 5 min window). |
| **Retries** | Implement exponential backoff (max 5 retries). |
| **Idempotency** | Provide `X-Webhook-Id` header; consumers must deduplicate. |
| **Payload Size** | Keep < 1 MB. For larger data, send a reference (e.g., download URL). |
| **Security** | Never log the shared secret or raw signature. |
| **Monitoring** | Log delivery attempts and failures; set up alerts. |
| **Documentation** | Publish webhook schema, signature method, and sample code. |

## 8. Complete C# Implementation Example

### 8.1 Provider: Registration & Sending Webhooks

**Program.cs (register services)**
```csharp
builder.Services.AddSingleton<IWebhookStore, InMemoryWebhookStore>();
builder.Services.AddSingleton<WebhookDeliveryService>();
builder.Services.AddHttpClient();
```

**WebhookDeliveryService (complete)**
```csharp
public class WebhookDeliveryService
{
    private readonly IHttpClientFactory _httpClientFactory;
    private readonly ILogger _logger;
    public WebhookDeliveryService(IHttpClientFactory factory, ILogger<WebhookDeliveryService> logger)
    {
        _httpClientFactory = factory;
        _logger = logger;
    }

    public async Task SendWebhookAsync(WebhookRegistration registration, object payload, int attempt = 1)
    {
        var jsonPayload = JsonSerializer.Serialize(payload);
        var timestamp = DateTimeOffset.UtcNow.ToUnixTimeSeconds().ToString();
        var signature = ComputeSignature(jsonPayload, timestamp, registration.SharedSecret);
        var webhookId = Guid.NewGuid().ToString();

        var client = _httpClientFactory.CreateClient();
        var request = new HttpRequestMessage(HttpMethod.Post, registration.Url)
        {
            Content = new StringContent(jsonPayload, Encoding.UTF8, "application/json")
        };
        request.Headers.Add("X-Webhook-Signature", $"sha256={signature}");
        request.Headers.Add("X-Webhook-Timestamp", timestamp);
        request.Headers.Add("X-Webhook-Id", webhookId);
        request.Headers.Add("User-Agent", "MyAPI-Webhook/1.0");

        try
        {
            var response = await client.SendAsync(request);
            if (response.IsSuccessStatusCode)
            {
                _logger.LogInformation("Webhook delivered to {Url}", registration.Url);
                return;
            }

            // Retry on 5xx or network errors
            if ((int)response.StatusCode >= 500 || response.StatusCode == HttpStatusCode.RequestTimeout)
            {
                await ScheduleRetry(registration, payload, attempt);
            }
            else
            {
                _logger.LogWarning("Permanent failure for webhook {Url}: {StatusCode}", registration.Url, response.StatusCode);
            }
        }
        catch (HttpRequestException ex)
        {
            _logger.LogError(ex, "Network error sending webhook to {Url}", registration.Url);
            await ScheduleRetry(registration, payload, attempt);
        }
    }

    private async Task ScheduleRetry(WebhookRegistration registration, object payload, int attempt)
    {
        const int maxRetries = 5;
        if (attempt >= maxRetries) return;
        var delay = TimeSpan.FromSeconds(Math.Pow(2, attempt));
        await Task.Delay(delay); // In production, use a persistent queue (Hangfire, RabbitMQ)
        await SendWebhookAsync(registration, payload, attempt + 1);
    }

    private string ComputeSignature(string payload, string timestamp, string secret)
    {
        var message = timestamp + "." + payload;
        using var hmac = new HMACSHA256(Encoding.UTF8.GetBytes(secret));
        var hash = hmac.ComputeHash(Encoding.UTF8.GetBytes(message));
        return Convert.ToBase64String(hash);
    }
}
```

### 8.2 Consumer: Webhook Receiver with Validation

**WebhookController.cs**
```csharp
[ApiController]
[Route("webhooks")]
public class WebhookReceiverController : ControllerBase
{
    private readonly IWebhookSecretStore _secretStore;
    private readonly IIdempotencyStore _idempotencyStore;
    private readonly IOrderProcessor _orderProcessor;

    [HttpPost("orders")]
    public async Task<IActionResult> ReceiveOrderWebhook()
    {
        // Validate headers
        if (!Request.Headers.TryGetValue("X-Webhook-Signature", out var sigHeader) ||
            !Request.Headers.TryGetValue("X-Webhook-Timestamp", out var tsHeader) ||
            !Request.Headers.TryGetValue("X-Webhook-Id", out var idHeader))
            return BadRequest("Missing required headers");

        var signature = sigHeader.ToString().Replace("sha256=", "");
        var timestamp = tsHeader.ToString();
        var webhookId = idHeader.ToString();

        // Idempotency check
        if (await _idempotencyStore.IsProcessedAsync(webhookId))
            return Ok();

        // Read raw body
        var rawBody = await ReadRawBodyAsync();

        // Retrieve secret (e.g., based on the URL path)
        var secret = await _secretStore.GetSecretForWebhookUrl(Request.Path);

        // Verify signature
        var expected = ComputeSignature(rawBody, timestamp, secret);
        if (!CryptographicOperations.FixedTimeEquals(
            Convert.FromBase64String(signature), 
            Convert.FromBase64String(expected)))
        {
            return Unauthorized("Invalid signature");
        }

        // Check timestamp (5 min tolerance)
        var now = DateTimeOffset.UtcNow.ToUnixTimeSeconds();
        if (Math.Abs(now - long.Parse(timestamp)) > 300)
            return BadRequest("Webhook too old");

        // Process payload
        var payload = JsonSerializer.Deserialize<OrderEvent>(rawBody);
        await _orderProcessor.ProcessAsync(payload);

        // Mark as processed
        await _idempotencyStore.MarkProcessedAsync(webhookId, TimeSpan.FromDays(7));
        return Ok();
    }

    private string ComputeSignature(string payload, string timestamp, string secret)
    {
        var message = timestamp + "." + payload;
        using var hmac = new HMACSHA256(Encoding.UTF8.GetBytes(secret));
        var hash = hmac.ComputeHash(Encoding.UTF8.GetBytes(message));
        return Convert.ToBase64String(hash);
    }

    private async Task<string> ReadRawBodyAsync()
    {
        Request.EnableBuffering();
        using var reader = new StreamReader(Request.Body, leaveOpen: true);
        var body = await reader.ReadToEndAsync();
        Request.Body.Position = 0;
        return body;
    }
}
```

## 9. Conclusion

Public webhooks are a powerful mechanism for real‑time integration, but they require careful security design. This guide covers:

- **Registration** with token‑based authentication and shared secret generation.
- **Secure delivery** using `X-Webhook-Signature` (HMAC-SHA256) and optional timestamp to prevent replay attacks.
- **Idempotency** using `X-Webhook-Id` to avoid duplicate processing.
- **Retries** with exponential backoff.
- **Best practices** including HTTPS, constant‑time comparison, and monitoring.

The provided C# implementations give you a solid foundation for both provider (sending webhooks) and consumer (receiving and validating webhooks) sides. Always test with real‑world scenarios and keep your secrets safe.

For further reading, see [RFC 2119](https://tools.ietf.org/html/rfc2119) and [Stripe’s webhook signature guide](https://stripe.com/docs/webhooks/signatures).
