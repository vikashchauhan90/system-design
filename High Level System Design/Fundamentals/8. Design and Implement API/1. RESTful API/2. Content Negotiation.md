# Comprehensive Guide to Content Negotiation for Web APIs

Content negotiation is the mechanism that allows a client and server to agree on the format of data being exchanged. It ensures that the server returns data in a format the client can process, and the client sends data in a format the server understands. This document covers the two primary headers—`Accept` and `Content-Type`—with examples for JSON, XML, file downloads, and custom formats, plus implementation guidance in ASP.NET Core.


## 1. Introduction to Content Negotiation

In a RESTful API, the same resource (e.g., `/api/products/123`) can be represented in multiple formats: JSON, XML, CSV, Protobuf, etc. The client indicates its preference via the `Accept` header; the server uses `Content-Type` to declare what it actually sends.

**Two directions:**
- **Request negotiation (client → server):** Client uses `Content-Type` to describe the body format.
- **Response negotiation (server → client):** Client uses `Accept` to list preferred response formats.

**HTTP headers involved:**
| Header          | Purpose                                                                 | Example                         |
|----------------|-------------------------------------------------------------------------|---------------------------------|
| `Accept`        | Lists media types the client can understand in the response.            | `Accept: application/json`      |
| `Content-Type`  | Indicates the media type of the request body (or response body).        | `Content-Type: application/xml` |
| `Accept-Language` | Language negotiation (e.g., `en-US`, `fr`).                           |                                 |
| `Accept-Encoding` | Compression (gzip, br).                                              |                                 |

This guide focuses on `Accept` and `Content-Type`.



## 2. The Accept Header (Client‑Side)

The `Accept` header is sent by the client to tell the server which response media types are acceptable. The server should return the best match based on its capabilities.

**Syntax:**
```
Accept: <MIME_type>, <MIME_type>;q=<quality_factor>
```
- `q` = weight (0 to 1), default 1. Higher = more preferred.
- Multiple types are comma‑separated.

**Examples:**
```
Accept: application/json
Accept: application/json, application/xml
Accept: application/xml;q=0.8, application/json;q=0.5
Accept: text/html, application/xhtml+xml, application/xml;q=0.9, */*;q=0.8
```

### Use Cases
- **API clients** that only understand JSON → `Accept: application/json`
- **Browser** requesting HTML → `Accept: text/html`
- **Data export** tool that can handle CSV or Excel → `Accept: text/csv, application/vnd.ms-excel`

### Tips for Clients
- Always send a meaningful `Accept` header; avoid `*/*` unless you handle any response.
- If you need a specific version (e.g., JSON API), use vendor MIME types: `application/vnd.myapi.v2+json`
- For file downloads, request the exact MIME type of the file (e.g., `image/png`).


## 3. The Content‑Type Header (Server‑Side)

The `Content-Type` header is used in two contexts:
1. **Request:** The client tells the server what format the request body is in.
2. **Response:** The server tells the client what format the response body is in.

**Syntax:**
```
Content-Type: <MIME_type>; charset=<character_set>
```

**Examples:**
```
Content-Type: application/json
Content-Type: application/xml; charset=utf-8
Content-Type: multipart/form-data; boundary=---boundary
```

### Use Cases
- **POST/PUT with JSON data** → `Content-Type: application/json`
- **Uploading a file** → `Content-Type: multipart/form-data`
- **Returning a CSV file** → `Content-Type: text/csv`

### Tips for Servers
- Always set `Content-Type` correctly; clients rely on it.
- Use the appropriate character set (usually UTF‑8).
- For file downloads, also send `Content-Disposition: attachment; filename="..."` to force download.



## 4. Common Media Types

| Format          | MIME Type                                  | Typical Use                      |
|----------------|--------------------------------------------|----------------------------------|
| JSON            | `application/json`                         | REST APIs, configuration         |
| XML             | `application/xml` or `text/xml`            | SOAP, legacy systems             |
| Plain Text      | `text/plain`                               | Logs, simple messages            |
| HTML            | `text/html`                                | Web pages, rich responses        |
| CSV             | `text/csv`                                 | Data export, spreadsheets        |
| PDF             | `application/pdf`                          | Reports, documents               |

| Form data       | `application/x-www-form-urlencoded`        | HTML forms, simple key/value     |
| Multipart       | `multipart/form-data`                      | File uploads, mixed content      |
| Protobuf        | `application/x-protobuf`                   | High‑performance RPC             |
| Custom vendor   | `application/vnd.mycompany.v3+json`        | Versioned APIs                   |



## 5. Implementing Content Negotiation in ASP.NET Core

ASP.NET Core has built‑in support for content negotiation through **formatters**. Output formatters handle response formatting; input formatters handle request body deserialization.

### 5.1 Default Formatters

The framework includes formatters for:
- **JSON** (`System.Text.Json` or Newtonsoft.Json)
- **XML** (requires `AddXmlSerializerFormatters()` or `AddXmlDataContractSerializerFormatters()`)
- **Plain text** (basic)

### 5.2 Configuration Example (Program.cs)

```csharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers(options =>
{
    // Optional: customise content negotiation
    options.RespectBrowserAcceptHeader = true; // true = use Accept header from browser
    options.ReturnHttpNotAcceptable = true;    // return 406 if no matching formatter
})
.AddXmlSerializerFormatters()                  // adds XML support (both in/out)
.AddJsonOptions(options =>                     // configure JSON behaviour
{
    options.JsonSerializerOptions.PropertyNamingPolicy = JsonNamingPolicy.CamelCase;
});
```

### 5.3 Enabling XML Formatters

XML is not included by default. Install the package:
```
dotnet add package Microsoft.AspNetCore.Mvc.Formatters.Xml
```

Then call `AddXmlSerializerFormatters()` or `AddXmlDataContractSerializerFormatters()`.

### 5.4 Custom Formatter Example (CSV)

You can create a custom output formatter to support formats like CSV.

```csharp
public class CsvOutputFormatter : TextOutputFormatter
{
    public CsvOutputFormatter()
    {
        SupportedMediaTypes.Add(MediaTypeHeaderValue.Parse("text/csv"));
        SupportedEncodings.Add(Encoding.UTF8);
        SupportedEncodings.Add(Encoding.Unicode);
    }

    protected override bool CanWriteType(Type type)
    {
        return typeof(IEnumerable<object>).IsAssignableFrom(type);
    }

    public override async Task WriteResponseBodyAsync(OutputFormatterWriteContext context, Encoding selectedEncoding)
    {
        var response = context.HttpContext.Response;
        var streamWriter = new StreamWriter(response.Body, selectedEncoding);
        
        var items = (IEnumerable<object>)context.Object;
        var properties = items.FirstOrDefault()?.GetType().GetProperties();
        
        if (properties != null)
        {
            // Write header
            await streamWriter.WriteLineAsync(string.Join(",", properties.Select(p => p.Name)));
            // Write rows
            foreach (var item in items)
            {
                var values = properties.Select(p => p.GetValue(item)?.ToString());
                await streamWriter.WriteLineAsync(string.Join(",", values));
            }
        }
        await streamWriter.FlushAsync();
    }
}
```

Register it:
```csharp
builder.Services.AddControllers(options =>
{
    options.OutputFormatters.Insert(0, new CsvOutputFormatter());
});
```


## 6. Examples: Returning Different Formats from an Action

### 6.1 JSON and XML (Automatic via Content Negotiation)

```csharp
[ApiController]
[Route("api/[controller]")]
public class ProductsController : ControllerBase
{
    [HttpGet("{id}")]
    public ActionResult<Product> GetProduct(int id)
    {
        var product = new Product { Id = id, Name = "Laptop", Price = 999.99 };
        // ASP.NET Core will automatically choose JSON or XML based on Accept header
        return Ok(product);
    }
}
```

- If client sends `Accept: application/json` → JSON.
- If client sends `Accept: application/xml` → XML.
- If client sends `Accept: text/csv` and a CSV formatter is registered → CSV.

### 6.2 Forcing a Specific Format with `[Produces]` Attribute

You can restrict what an action can produce:

```csharp
[HttpGet("{id}")]
[Produces("application/json")]  // only JSON allowed
public ActionResult<Product> GetProduct(int id) { ... }
```

Or allow multiple but set a default:

```csharp
[HttpGet("{id}")]
[Produces("application/json", "application/xml")]
public ActionResult<Product> GetProduct(int id) { ... }
```

### 6.3 Returning a File (PDF, Image, CSV)

Use `FileResult` to return binary files or raw bytes.

```csharp
[HttpGet("report")]
public IActionResult GetReport()
{
    byte[] pdfBytes = GeneratePdf(); // your PDF generation logic
    return File(pdfBytes, "application/pdf", "report.pdf");
}

[HttpGet("image/{id}")]
public IActionResult GetImage(int id)
{
    var imageBytes = System.IO.File.ReadAllBytes($"images/{id}.png");
    return File(imageBytes, "image/png");
}
```

The `Content-Type` header is set automatically from the second parameter.

### 6.4 Handling Accept Header for File Downloads

If you want to return different file formats based on `Accept`:

```csharp
[HttpGet("export")]
public IActionResult ExportData()
{
    var accept = Request.Headers["Accept"].ToString();
    if (accept.Contains("text/csv"))
    {
        var csvData = GenerateCsv();
        return File(Encoding.UTF8.GetBytes(csvData), "text/csv", "data.csv");
    }
    else if (accept.Contains("application/json"))
    {
        var jsonData = JsonSerializer.Serialize(GetData());
        return Ok(jsonData);
    }
    else
    {
        return BadRequest("Unsupported format");
    }
}
```

## 7. Content Negotiation in Requests (Client Sending Data)

When the client sends data (POST, PUT, PATCH), the `Content-Type` header must match the format used in the body.

### 7.1 JSON Request

```
POST /api/products HTTP/1.1
Content-Type: application/json

{"name":"Laptop","price":999.99}
```

ASP.NET Core model binding will automatically deserialize using the JSON input formatter.

### 7.2 XML Request

```
POST /api/products HTTP/1.1
Content-Type: application/xml

<Product><Name>Laptop</Name><Price>999.99</Price></Product>
```

Requires XML formatters to be added.

### 7.3 Form URL‑Encoded (traditional web form)

```
POST /api/products HTTP/1.1
Content-Type: application/x-www-form-urlencoded

name=Laptop&price=999.99
```

In ASP.NET Core, you can use `[FromForm]` to bind to a model.

### 7.4 Multipart Form Data (file upload)

```csharp
[HttpPost("upload")]
public async Task<IActionResult> UploadFile(IFormFile file)
{
    // file is automatically bound from multipart/form-data
    return Ok(new { fileName = file.FileName, size = file.Length });
}
```

Client sends:
```
POST /api/upload HTTP/1.1
Content-Type: multipart/form-data; boundary=---boundary
```


## 8. Advanced Content Negotiation Techniques

### 8.1 Custom Media Types for Versioning

Use vendor‑specific media types to version your API:

```
Accept: application/vnd.myapi.v2+json
```

In your action, read the `Accept` header and route accordingly.

### 8.2 Language Negotiation with `Accept-Language`

Return localized messages:

```csharp
[HttpGet("message")]
public IActionResult GetMessage()
{
    var language = Request.Headers["Accept-Language"].ToString();
    var message = language.StartsWith("fr") ? "Bonjour" : "Hello";
    return Ok(new { message });
}
```

### 8.3 Handling `406 Not Acceptable`

When the server cannot produce any of the media types listed in the `Accept` header, it should return `406 Not Acceptable`. Enable this with:

```csharp
options.ReturnHttpNotAcceptable = true;
```


## 9. Best Practices and Tips

### For API Designers
1. **Support JSON by default** – it’s the universal language of web APIs.
2. **Add XML only if needed** – many clients prefer JSON; XML adds overhead.
3. **Use standard MIME types** – avoid inventing custom types unless necessary.
4. **Respect the Accept header** – do not ignore it; return the best match.
5. **Set `Content-Type` correctly** – always include it in responses.
6. **For file downloads, also send `Content-Disposition`** to suggest a filename.
7. **Return 406** when you cannot satisfy the `Accept` header – it’s more correct than sending a default format.

### For API Clients
1. **Always send an explicit `Accept` header** – avoid relying on defaults.
2. **Use quality values (`q`)** to indicate preference when multiple formats are acceptable.
3. **Check the response `Content-Type`** – don’t assume it’s what you requested.
4. **For file downloads, inspect `Content-Disposition`** to get the filename.

### Security Considerations
- Validate `Content-Type` of requests to prevent cross‑site scripting (XSS) via unexpected content.
- Be careful with binary uploads – limit file size and scan for malware.
- When using custom formatters, ensure they don’t expose sensitive data.


## 10. Complete Example: API with JSON, XML, CSV, and File Download

```csharp
[ApiController]
[Route("api/reports")]
public class ReportsController : ControllerBase
{
    private readonly List<Product> _products = new()
    {
        new Product { Id = 1, Name = "Laptop", Price = 999.99 },
        new Product { Id = 2, Name = "Mouse", Price = 19.99 }
    };

    // GET /api/reports/products
    // Accept: application/json -> JSON
    // Accept: application/xml  -> XML
    // Accept: text/csv         -> CSV (if custom formatter added)
    [HttpGet("products")]
    public IActionResult GetProducts()
    {
        return Ok(_products);
    }

    // GET /api/reports/download?format=csv
    // Manual format selection via query parameter
    [HttpGet("download")]
    public IActionResult DownloadReport(string format)
    {
        if (format == "csv")
        {
            var csv = "Id,Name,Price\n" + string.Join("\n", _products.Select(p => $"{p.Id},{p.Name},{p.Price}"));
            return File(Encoding.UTF8.GetBytes(csv), "text/csv", "products.csv");
        }
        else if (format == "pdf")
        {
            byte[] pdf = GeneratePdfReport(); // your method
            return File(pdf, "application/pdf", "report.pdf");
        }
        else
        {
            return BadRequest("Supported formats: csv, pdf");
        }
    }

    private byte[] GeneratePdfReport() => System.IO.File.ReadAllBytes("template.pdf");
}
```

## 12. Final Notes

- Content negotiation is a key pillar of RESTful API design.
- JSON should be your default format; add others on demand.
- Use the built‑in formatters in ASP.NET Core for JSON and XML; extend with custom formatters for niche formats.
- Always validate and set the correct `Content-Type` to avoid misinterpretation.
- For file downloads, leverage `FileResult` and set `Content-Disposition` to control client behaviour.
